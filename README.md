# Presentation-Py

Инструмент для автоматической генерации презентаций PowerPoint на основе данных о спринтах разработки. Создает структурированные слайды с информацией об эпиках, задачах и достижениях команды.

## Установка

Убедитесь, что у вас установлен [uv](https://github.com/astral-sh/uv) для управления зависимостями.

```bash
uv init
uv add python-pptx
uv sync
```

## Использование

### Запуск проекта

```bash
uv run main.py
```

По умолчанию создается презентация для спринта №1.

### Создание презентации для конкретного спринта

```bash
uv run main.py <номер_спринта> <use_ai>
```

Где:
- `<номер_спринта>` - номер спринта (например, 25)
- `<use_ai>` - режим анализа данных:
  - `none` - без ИИ, использует готовые обработанные данные из `data/data_<номер>.py`
  - `giga` - анализ через GigaChat (требует API ключа)
  - `lmstudio` - анализ через локальную модель LMStudio
  - `ollama` - анализ через локальную модель Ollama (deepseek-r1:8b)

#### Режимы с использованием ИИ
Читают PDF-файл спринта, анализируют данные через выбранный ИИ и создают презентацию.

Примеры:
```bash
# С использованием GigaChat
uv run main.py 25 giga

# С использованием LMStudio
uv run main.py 25 lmstudio

# С использованием Ollama
uv run main.py 25 ollama
```

#### Режим без использования ИИ
```bash
uv run main.py 25 none
```
Использует готовые обработанные данные из файла `data/data_<номер>.py` для создания презентации.

Результат сохраняется в папку `result/` как `sprint_<номер>.pptx`.

## Структура проекта

- `main.py` - основной скрипт генерации презентации
- `data.py` - данные для слайдов (эпики, задачи)
- `template/base.pptx` - базовый шаблон презентации
- `images/logo.png` - логотип (опционально)
- `result/` - папка для готовых презентаций

## Зависимости

- Python >= 3.12
- python-pptx >= 1.0.2
